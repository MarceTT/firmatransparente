## Compact Rules

### Next.js 15
- App Router only, no pages/
- Server Components by default, 'use client' only when needed
- Server Actions for mutations, no API routes for forms
- Dynamic params: `{ params }: { params: Promise<{ id: string }> }` — MUST await
- Metadata: export async function generateMetadata()
- Loading: loading.tsx for Suspense boundaries
- Error: error.tsx with 'use client'

### React 19
- NO useMemo, NO useCallback — React Compiler handles it
- use() hook for promises and context
- useActionState for form state (replaces useFormState)
- useOptimistic for optimistic updates
- ref is a regular prop now, no forwardRef needed
- Cleanup functions in refs: `ref={(el) => { setup(el); return () => cleanup(); }}`

### Tailwind 4
- @theme in CSS, not tailwind.config.js
- CSS variables: --color-primary, --spacing-lg
- NO var() in className — only Tailwind classes
- Use cn() from lib/utils for conditional classes
- Container queries: @container, @lg:container

### Zustand 5
- NO shallow — use useShallow from 'zustand/shallow'
- createStore for vanilla, create for React
- Middleware: devtools wraps others `devtools(persist(...))`
- immer for nested state: `set(produce(state => { state.x = y }))`

### Zod 4
- z.string().email(), z.coerce.number()
- z.input<typeof schema> for input types
- z.output<typeof schema> for parsed types
- Async: z.string().refine(async (val) => ...)

### TypeScript
- Strict mode always
- No any — use unknown + type guards
- Prefer interfaces for objects, types for unions
- Explicit return types on exported functions

### Playwright
- Page Object Model for complex flows
- data-testid for selectors
- await expect(locator).toBeVisible() before interactions
- Use test.describe for grouping
